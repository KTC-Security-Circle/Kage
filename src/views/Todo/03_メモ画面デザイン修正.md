# 03. メモ画面デザイン修正

## 現状の問題点

src/views_new/memos/view.py のデザインが、テンプレートの MemosScreen.tsx と以下の点で乖離している：

### 1. ヘッダーレイアウト

**テンプレート**：
- 固定ヘッダーエリア（ページタイトル、説明、検索バー、新規作成ボタン）
- 右上に検索バー（幅264px）とボタンの適切な配置
- 背景色とボーダーによる明確な区分

**現状**：
- 基本的なコンポーネント配置
- 検索バーがアクションバーに埋込み
- 視覚的階層が不明確

### 2. タブデザイン

**テンプレート**：
- 4つのタブ（Inbox、アクティブ、アイデア、アーカイブ）
- 各タブにアイコンとバッジカウント
- グリッドレイアウトによる均等配置
- アクティブタブの明確な視覚表現

**現状**：
- カスタムタブコンポーネント使用
- バッジカウントが正しく表示されない可能性
- スタイリングがテンプレートと異なる

### 3. メイン2ペインレイアウト

**テンプレート**：
- 左側：メモリスト（2/5の幅）
- 右側：詳細パネル（3/5の幅）
- リストと詳細の明確な分離
- 適切なスクロール制御

**現状**：
- 固定幅（400px）のリスト
- レスポンシブ対応が不十分
- スクロール領域の制御が不完全

### 4. メモカードデザイン

**テンプレート**：
- 統一されたカードデザイン
- AIステータスバッジ（利用可能、処理中、確認済み等）
- タグ表示（カラー付き）
- 作成日表示
- ホバーエフェクトと選択状態の視覚化

**現状**：
- 基本的なカード表示
- AIステータスバッジが簡素
- 選択状態の表現が不十分

### 5. 詳細パネル

**テンプレート**：
- メモ未選択時の空状態表示
- 選択時の詳細表示（タイトル、ステータス、内容、タグ、メタデータ）
- AI提案ボタン（状態による表示切り替え）
- 編集・削除ボタン

**現状**：
- 基本的な詳細表示実装済み
- UI の洗練度が不足
- アクションボタンのレイアウト改善必要
